## 3. Architecture

Deliberately layered so the same core can deploy as a SMART-on-FHIR app:
1. **CDS_CONFIG** — all clinical limits/bands, per concentration (U-100 Standard, U-10 Dilute).
2. **Engine** (`computeDose`, `validateParameter`, `roundTo`, `round2`) — pure functions; returns everything the UI/webhook needs.
3. **StandaloneParameterSource** — on-device param load/save (under SMART: a FHIR MedicationRequest maps to the same params shape; stub present).
4. **Hooks** — event bus + optional webhook dispatcher (no-op standalone).
5. **UI** — render functions; settings modal; order card; calculate gate.

Two concentrations: **U-100 Standard** (dosed in Units) and **U-10 Dilute** (dosed in Syringe Units, fixed 1-unit rounding). Age bands: `lt5 / a5_11 / gte12` (U-100), `infant` (U-10).

## 4. Current behavior (what's live at v1.7.2)

- **Rounding = NEAREST increment, standard rounding, ties round UP**. `roundTo(v,inc)=Math.floor(v*f+0.5+1e-9)/f`, `f=Math.round(1/inc)`. e.g. 2.24→2 both; 2.34→2 @1, →2.5 @0.5; ties 2.5→3 @1, 2.25→2.5 @0.5. A dose that rounds to 0 is shown as 0 (no nonzero floor — math, not hypo logic).
- **Calculate gate**: total dose + all in-panel alerts hidden until **both** Current BG and carbs entered **AND** a centered **"Calculate Dose"** button is tapped. Editing any input / IOB / saving settings invalidates a shown dose (button returns). `renderAdvisories()` is a no-op — every alert lives inside the dose panel, so gating the panel covers them.
- **Sub-calc previews** (carb coverage; correction) under each field **commit on blur, not per keystroke**. `renderSubcalcs()` (blur/Calculate/ack/save) is separate from `recompute()` (the live gate). Typing a field clears its own preview until commit.
- **Prandial computed as soon as carbs is present** (before the both-inputs gate) so the carb-coverage preview is correct with BG still empty.
- **Order card** (top): repurposed from the old meal picker into a **display-only live order detail** — bold product `orderName` + a dynamically built sig (`orderSig()`): `1 dose subcutaneously {schedule}, Insulin/carb ratio 1:{ic}, Corr factor: {cf}, Target BG: {target}, Round to nearest {0.5 unit | 1 Syringe Unit}, Calculate dose to correct BG and cover carbs.` Chevron hidden; default cue kept only for the (ignored) icon. Rebuilt on every save.
- **Header subtitle**: `STANDARD U-100` / gold `**DILUTE U-10**`.
- **IOB lookback = 2 h** (nurse-attested checkbox; no time computation).
- **Caution badges** (order params): green check, else amber ⚠ caution when I:C < age soft-low or ≥100; CF < age soft-low (lt5/a5_11 only) or >150 (verify) or ≥200 (high warn, all ages); Target outside age band. Target off-tier is **reconcile-and-proceed** (challenge → prescriber-verified checkbox in the dose panel computes the dose), not a hard block.
- **BG entry 1–2000**; >600 → correction uses 600 (live amber notice); BG=0 critical branch is dead code (entry min 1). Carbs entry clamps to `entryMax`; >formulaCap (250) capped with notice.
- **Deployment notes** ("FHIR SMART & webhooks") and the **result-webhook** field both live inside **Settings**.
- **Order-param gauges** in Settings: interactive sliders with zone backgrounds, median (M) ring, mode dots, pixel-snap magnet. `DIST` detentes + clinical-contender medians **verified by Troy** (v1.8.0). The big-M circle sits on the strongest clinical value, not always the statistical median (which stays mathematically correct under the hood).
